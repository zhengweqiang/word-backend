# 视频上传自动可预览和发布设计

## 目标

管理员和教师成功上传视频后，视频立即以“可预览”和“已发布”状态保存。教师登录视频资源页面时，能够立即看到管理员上传的视频并预览。

## 问题根因

管理员上传的视频被标记为系统视频。教师只会看到同时满足“系统视频、可预览、已发布且具有播放地址”的系统视频；当前上传流程会依据云端返回状态决定是否可预览，并默认设为未发布。因此管理员刚上传的视频通常不符合教师的视频库可见条件。

## 范围

- 仅修改新的、上传成功的视频记录。
- ADMIN 和 TEACHER 的上传结果均保存为 `READY` 与 `PUBLISHED`，并记录 `publishedAt`。
- 上传过程中发生异常时，仍保留现有失败处理，不伪造可预览或已发布状态。
- 教师对其他教师私有视频的管理权限不变；教师仅获得管理员系统视频的查看和预览能力，不获得管理、发布、删除权限。
- 已存在的视频记录与手动云端同步行为不在本次修改范围内。

## 设计

上传服务在云端上传成功、媒体地址已写入视频实体后，统一将新视频的 `status` 设为 `READY`、`cloudPublishStatus` 设为 `PUBLISHED`，并将 `publishedAt` 设置为当前时间。管理员上传的视频仍保留 `SYSTEM` 范围，教师上传的视频仍保留 `TEACHER` 范围。

无需修改教师列表的可见性规则：管理员视频满足既有的系统视频可见条件后，会自然出现在教师的视频库中。现有管理权限计算继续限制教师只能管理自己的教师视频。

## 验收与测试

1. ADMIN 上传成功后，响应状态为 `READY`、云端发布状态为 `PUBLISHED`，并带有发布时间。
2. TEACHER 上传成功后，响应状态为 `READY`、云端发布状态为 `PUBLISHED`，并带有发布时间。
3. TEACHER 查询视频库时，能看到管理员上传且状态为 `READY`、`PUBLISHED` 的系统视频。
4. TEACHER 对该管理员系统视频仍无管理权限。
5. 既有视频服务测试保持通过。
