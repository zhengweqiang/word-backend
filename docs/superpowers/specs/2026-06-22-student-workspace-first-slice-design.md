# 学生工作台首个纵向切片设计

## 目标

把已验证的学生工作台原型接入生产前后端，让学生登录后可以查看跨计划聚合的今日任务、完成真实的逐词学习，并在学习卡中使用音节拆读和朗读能力。教师端、管理员端和现有计划管理能力保持不变。

## 本期范围

- 学生登录后进入独立 `StudentWorkspace`，教师继续使用现有工作区，管理员继续跳转 `/admin/`。
- 首页展示真实今日任务摘要和计划提醒。
- 多个进行中计划合并为一个今日学习队列。
- 学习卡支持“我会了”“再学一次”“先跳过”、注意力记录和音节拆读。
- 现有学生词书浏览能力迁入“词库”。
- “我的”展示只读账户资料和退出登录。
- 手机使用底部导航和单列布局；桌面使用左侧导航和响应式双栏。
- 管理员可手动触发当前已发布计划缺失音节数据的 AI 批量补全。

## 明确延期

- 自由学习。
- 全局卡片盒、自动错词本和收藏夹。
- 正式考试待办、自测和考试分轨改造。
- 30 天日历与分析统计。
- 学生资料编辑和密码修改。

这些入口在对应能力上线前不出现在工作台中，不使用演示数量或禁用占位按钮。

## 工作台信息架构

### 首页

- 今日任务总数、已完成数和完成率。
- 逾期复习、今日复习和新词数量。
- “继续今日学习”主操作。
- 仅展示可闭环的提醒：今日任务未完成、存在逾期复习。

空状态分为：

- `尚未安排学习任务`：没有有效的进行中计划或今日任务。
- `今日任务已完成`：存在今日任务且队列已清空。

### 学习

- 显示统一队列的当前任务、计划名称、任务类型、阶段、单词、音标、释义和停留时长。
- 同一个单词出现在多个计划时按计划任务分别展示并分别提交。
- 队列按任务类型排序：逾期复习、今日复习、新词。
- 同一类型按应复习日期、计划发布时间、任务 ID 排序，保证稳定且可解释。

### 词库

- 仅展示当前学生已分配词书。
- 保留词书选取、单词列表、搜索和详情浏览。
- 不展示创建、导入、分配、考试创建等管理操作。

### 我的

- 展示显示名、用户名、角色、班级和教师信息（有数据时）。
- 支持退出登录。
- 不提供资料编辑或密码修改入口。

## 学习结果语义

- `我会了`：记录 `CORRECT`，完成当前计划任务并从队列移除。
- `先跳过`：记录 `SKIPPED`，完成当前计划任务并从队列移除。
- `再学一次`：记录 `INCORRECT`，不完成当前计划任务，将任务放到统一队列末尾。
- `再学一次`不设硬上限，卡片显示本轮尝试次数；学生可通过“我会了”或“先跳过”结束。
- 每次提交后由服务端重新返回权威工作台状态，前端不推测计划完成数。

## 聚合接口

新增：

- `GET /api/students/me/dashboard`
- `POST /api/students/me/dashboard/records`

`GET` 返回：

- `taskDate`
- `hasPlans`
- `allTasksCompleted`
- `overdueCount`
- `reviewCount`
- `newCount`
- `completedCount`
- `totalCount`
- `completionRate`
- `reminders`
- `queue`

每个队列项包含：

- `studentStudyPlanId`
- `studyDayTaskItemId`
- `planName`
- `planPublishedAt`
- `metaWordId`
- `word`
- `translation`
- `phonetic`
- `phoneticDetail`
- `syllableDetail`
- `taskType`
- `phase`
- `dueDate`
- `attemptCount`

`POST` 请求包含现有注意力字段，并额外携带 `studentStudyPlanId`。服务端校验任务属于当前学生；响应返回更新后的完整 dashboard。

## 音节数据

`meta_words` 新增 JSONB 字段 `syllable_detail`。结构支持不依赖口音的拼写段和英美两套发音：

```json
{
  "segments": [
    {
      "text": "re",
      "ukPhonetic": "/rɪ/",
      "usPhonetic": "/rɪ/",
      "ukAudioUrl": null,
      "usAudioUrl": null
    }
  ]
}
```

- 有音频 URL 时优先播放真人音频。
- 无音频时回退 `speechSynthesis`。
- 英美音切换保存在浏览器本机，默认美音。
- 音节数据缺失时隐藏拆读区，普通学习卡继续可用。
- 词书导入和 AI 单词生成格式同步支持 `syllableDetail`。

## 音节批量补全

- 管理员手动触发，只扫描当前已发布计划涉及且 `syllable_detail` 为空的单词。
- AI 结果必须通过结构校验：音节数组非空、音节文本非空、按顺序拼接并标准化后等于原词。
- 合格结果自动入库；失败项保留单词 ID、单词和失败原因，供管理员重试或人工修正。
- 任务不自动在应用启动时运行，不阻塞学生工作台。

## 前端组件边界

- `StudentWorkspace`：学生端页面状态和响应式外壳。
- `StudentDashboardHome`：首页摘要、提醒和空状态。
- `StudentStudySession`：当前任务、注意力采集和结果提交。
- `SyllableReader`：口音选择、音频播放、语音回退和播放状态。
- `StudentLibrary`：复用现有学生词书浏览能力。
- `StudentProfile`：只读资料和退出登录。

`App.tsx` 只负责认证、角色分流和教师现有工作区，不继续承载学生页面细节。

## 错误与恢复

- dashboard 加载失败显示带“重试”的错误状态。
- 学习结果提交期间禁用三个结果按钮，防止重复提交。
- 提交失败保留当前词和计时信息，允许重试。
- 音频失败降级到浏览器语音；两者都不可用时显示非阻塞提示。
- 音节批量补全逐词隔离失败，不因单个词终止整批任务。

## 验收标准

- 学生登录后默认进入工作台，教师和管理员行为不回归。
- 多计划任务聚合、排序和重复词分计划展示符合规则。
- `INCORRECT`记录后任务仍未完成并移动到队尾；`CORRECT`和`SKIPPED`完成任务。
- 首页计数在每次提交后由服务端状态更新。
- 音节拆读支持英美音、真人音频优先和语音合成回退。
- 缺少音节数据的词仍可完成学习。
- 管理员批量补全只处理已发布计划相关缺失词，并拒绝拼接不还原原词的数据。
- 390x844、320x740 和桌面视口无横向溢出或交互遮挡。
- 后端测试、前端构建与 lint 通过。
- 修改 `frontend/` 后按仓库要求重建并重启 Docker `frontend` 服务。
